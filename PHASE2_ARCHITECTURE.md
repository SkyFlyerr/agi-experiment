## Scalability Considerations

- **Connection Pool**: 10 concurrent database connections
- **Async Processing**: Media downloads don't block webhook responses
- **FastAPI**: Handles concurrent requests efficiently
- **Job Queue**: Reactive jobs processed asynchronously by worker
- **Media Storage**: Local filesystem (migrate to MinIO/S3 for production)

## Security Layers

1. **Webhook Secret**: X-Telegram-Bot-Api-Secret-Token verification
2. **HTML Escaping**: All user text escaped before sending
3. **SQL Injection**: Parameterized queries (asyncpg)
4. **File Isolation**: Media stored in dedicated directory
5. **Raw Payload**: Debugging without exposing sensitive data in logs
